하나의 RS485 채널에는 최대 32개의 슬레이브 장치를 연결할 수 있습니다. 슬레이브 장치에 연결하려면 먼저 해당 장치를 마스터 장치에 등록해야 합니다.

## Config

```protobuf
message RS485Config {
  repeated RS485Channel channels;
  IntelligentPDInfo intelligentInfo;
}
```
{: #RS485Config }

[channels](#RS485Channel)
: CoreStation은 4개의 RS485 채널을 가지고 있습니다. 그 외의 장치들은 하나의 채널을 가집니다.

[intelligentInfo](#IntelligentPDInfo)
: 일부 BioStar 장치는 OSDP 프로토콜을 사용하여 타사 ACU에 연결할 수 있습니다. 이러한 경우 추가 옵션을 구성할 수 있습니다. 자세한 내용은 [해당 문서](https://support.supremainc.com/en/support/solutions/articles/24000066085)를 참고하십시오. 

```protobuf
message RS485Channel {
  uint32 channelID;
  Mode mode;
  uint32 baudRate;
  repeated SlaveDeviceInfo slaveDevices;
  bool useResistance;
  ChannelType channelType;
}
```
{: #RS485Channel }

[mode](#Mode)
: 

baudRate
: 사용 가능한 옵션은 9600, 19200, 38400, 57600, 115200입니다. 기본값은 115200입니다.

[slaveDevices](#SlaveDeviceInfo)
: 동일한 채널에 연결된 슬레이브 장치들을 나열합니다.

useResistance
: 저항 플래그 - 동작에는 영향을 미치지 않습니다.

[channelType](#ChannelType)
: RS485 채널 내의 슬레이브 장치들이 어떤 유형의 연결을 가지는지에 대한 정보입니다.

```protobuf
enum ChannelType {
  CHANNEL_TYPE_NORMAL = 0;
  CHANNEL_TYPE_SUPREMA = 1;
  CHANNEL_TYPE_OSDP_STANDARD = 2;
  CHANNEL_TYPE_OSDP_STANDARD_FULL = 3;
}
```
{: #ChannelType }

CHANNEL_TYPE_NORMAL
: 기본값입니다. 연결된 슬레이브가 없어 채널 유형이 지정되지 않습니다.

CHANNEL_TYPE_SUPREMA
: Suprema 채널 유형입니다.

CHANNEL_TYPE_OSDP_STANDARD
: OSDP 채널 유형입니다.

CHANNEL_TYPE_OSDP_STANDARD_FULL
: OSDP 연결이 더 이상 불가능합니다.

```protobuf
enum Mode {
  NOT_USE = 0;
  MASTER = 1;
  SLAVE = 2;
  STANDALONE = 3;
}
```
{: #Mode }
MASTER
: MASTER 모드인 장치만 슬레이브 장치를 가질 수 있습니다.

SLAVE
: SLAVE 모드인 장치만 다른 장치에 슬레이브로 추가될 수 있습니다.

STANDALONE
: 기본 모드입니다. 

```protobuf
message IntelligentPDInfo {
  bool useExceptionCode;
  bytes exceptionCode;
  IPDOutputFormat outputFormat;
  uint32 OSDPID;
  bool supportConfig;
}
```
{: #IntelligentPDInfo }

useExceptionCode
: true인 경우, 인증에 실패하면 __exceptionCode__ 를 타사 ACU로 전송합니다.

exceptionCode
: 인증에 실패했을 때 타사 ACU로 전송할 8바이트 코드입니다.

[outputFormat](#IPDOutputFormat)
: 인증에 성공했을 때 타사 ACU로 전송할 데이터입니다.

OSDPID
: OSDP 네트워크의 고유 식별자입니다.

supportConfig
: 이 값이 0이면 장치는 Intelligent PD(Peripheral Device)와 관련된 모든 설정을 무시합니다.

```protobuf
enum IPDOutputFormat {
  CARDID = 0;
  USERID = 1;
}
```
{: #IPDOutputFormat}
CARDID
: 인증에 성공했을 때 카드 ID를 전송합니다.

USERID
: 인증에 성공했을 때 사용자 ID를 전송합니다.

## Slave devices

RS485 채널의 슬레이브 장치를 검색, 추가, 삭제할 수 있습니다.

```protobuf
message SlaveDeviceInfo {
  uint32 deviceID;
  device.Type type;
  bool enabled;
  bool connected;
  uint32 channelID;
  uint32 parentID;
}
```
{: #SlaveDeviceInfo }

deviceID
: RS485 채널에 등록된 슬레이브 장치의 ID입니다.

type
: 장치 모델 정보입니다.

enable
: 슬레이브 장치를 마스터에 연결하려면 true로 설정합니다.

connected
: 슬레이브 장치가 마스터에 연결되어 있는지 여부입니다.

channelID
: 슬레이브 장치 그룹을 분류하는 채널 ID 번호입니다.

parentID
: DI-24 하위의 카드 리더기가 자신의 마스터가 누구인지 나타내기 위한 정보입니다.

### SearchDevice

장치의 슬레이브를 검색합니다. 장치가 둘 이상의 RS485 채널을 가지고 있는 경우, 모든 채널이 동시에 탐색됩니다. 발견된 장치에 접근하려면 [SetDevice](#setdevice)를 사용하여 등록해야 합니다.

| Request |

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| deviceID | uint32 | 슬레이브 장치를 검색할 장치의 ID |

| Response |

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| slaveInfos |[SlaveDeviceInfo[]](#SlaveDeviceInfo) | RS485 채널에서 발견된 슬레이브 장치들 |

### GetDevice

등록된 슬레이브 장치를 가져옵니다.

| Request |

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| deviceID | uint32 | 장치의 ID |

| Response |

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| slaveInfos |[SlaveDeviceInfo[]](#SlaveDeviceInfo) | 장치에 등록된 슬레이브 장치들 |

### SetDevice

슬레이브 장치를 등록합니다. 

슬레이브는 한 번만 등록하면 됩니다. 그러나 이 정보는 장치 게이트웨이에 저장되지 않으므로, 장치 게이트웨이가 다시 연결될 때는 [Connect.SetSlaveDevice]({{'/api/connect/' | relative_url}}#setslavedevice) 또는 [ConnectMaster.SetSlaveDevice]({{'/api/connectMaster/' | relative_url}}#setslavedevice)를 사용해야 합니다.
{: .notice--warning}

| Request |

| Parameter | Type | Description |
| --------- | ---- | ----------- |
| deviceID | uint32 | 장치의 ID |
| slaveInfos |[SlaveDeviceInfo[]](#SlaveDeviceInfo) | 장치에 등록할 슬레이브 장치들 |
